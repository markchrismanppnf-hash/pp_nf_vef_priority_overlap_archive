## Core Principles

### Principle 1: Observable Facts Only, No Causal Claims

All claims in this archive describe **observable facts**, not attributions:

**Observable facts:**
- Both frameworks are named VEF
- DOI timestamps are immutable
- Structural parallels exist in comparable domains
- Global repository indexing makes frameworks discoverable

**NOT factual claims:**
- Whether Mudassir actually discovered your work
- Whether copying or plagiarism occurred
- Whether attribution is owed
- What consequences should follow

### Principle 2: VEF Naming as Primary Evidence

The **exact VEF naming collision** is the primary discovery factor:
- Both frameworks explicitly use acronym "VEF"
- This increases discoverability probability to ~95%+ (if either researcher searches by framework name)
- This is objective, mathematically grounded evidence
- It does NOT prove discovery occurred; it documents high likelihood if searched

### Principle 3: Secondary Factors Are Supplementary

Additional discovery factors (M.M. initials, topic overlap, indexing) are **supplementary**:
- M.M. initials: +~10% discovery likelihood
- Topic overlap: +~5%
- Global indexing: +~5%
- Combined: ~20% additional likelihood beyond VEF naming

**Framing:** VEF is the primary mechanism; other factors increase likelihood further.

### Principle 4: Immutable Digital Records as Ground Truth

All chronological claims rest on permanent DOI timestamps:
- Zenodo DOIs: CERN-operated, permanent, institutional commitment
- Figshare DOIs: Digital Science, permanent, institutional commitment
- OpenAIRE: EU infrastructure, permanently indexed

**Verification:** Any claim can be independently verified via direct DOI lookup.

## Documentation Standards

### Standard 1: Prioritize VEF Naming in Presentation

When discussing discovery factors:
1. **Lead with VEF collision** (primary, ~90% weight)
2. **Then mention M.M. initials** (supplementary, ~5% weight)
3. **Then mention topic overlap** (supplementary, ~3% weight)
4. **Then mention indexing** (supplementary, ~2% weight)

**Example:**
```
"The critical discovery factor is the exact VEF naming collision—both frameworks are 
explicitly named Volumetric Exchange Framework (VEF). This creates ~95%+ discoverability 
if either researcher searches by framework acronym. Additionally, M.M. initials provide 
a secondary collision point, and topic overlap (substrate, dark matter) provides further 
increased likelihood of discovery if related searches occur."
```

### Standard 2: Observable vs. Speculative Language

Tag each statement:

**[Observable]:** Facts currently verifiable
- "Your VEF framework exists on Zenodo"
- "Both frameworks are named VEF"
- "DOI timestamps are immutable"

**[Documented]:** Immutable records
- "Your DOI: 10.5281/zenodo.18615908"
- "Mudassir v5: ~September 2025"

**[Inferred]:** Logical deduction
- "Both frameworks solve identical singularity problem"
- "Discovery probability is high if framework acronym is searched"

**[Speculative]:** Possible but undocumented
- "Mudassir may have discovered your framework"
- "Knowledge transfer might have occurred"

### Standard 3: Similarity Indices Without Causation Claims

Structural similarity scores (0–100) are **descriptive** of mechanical equivalence, not **causal**:

```
Similarity Index ≠ Proof of causation

Similarity Index MEANS:
  - Frameworks use mechanically equivalent approaches
  - Problem space and solution strategy overlap significantly
  - IF knowledge transfer occurred, it would explain similarities
  - Patterns are worth documenting

Similarity Index DOES NOT MEAN:
  - One framework derived from the other
  - Intentional copying occurred
  - Attribution is owed
  - Misconduct is likely
```

### For Readers: DIY Verification of VEF Naming

**Protocol: Confirm Exact VEF Naming Collision**

```

  1. Visit: https://doi.org/10.5281/zenodo.18615908
  2. Confirm: Full title explicitly includes "Volumetric Exchange Framework (VEF)"
  3. Record: Exact naming, publication date, DOI

Step 2: Verify Parallel Framework Uses VEF
  1. Obtain: Full text of Mudassir v4 and v5 papers
  2. Search: For explicit use of acronym "VEF" or "Volumetric Exchange"
  3. Record: Exact context, whether attributed to Mark Chrisman
  4. If not found: Naming collision is partial or non-existent

Step 3: Verify Search Collision
  1. Visit: https://zenodo.org
  2. Search: "VEF" + "substrate"
  3. Record: Whether both frameworks appear in results
  4. Note: Ordering and ranking of results
```

## Limitations Acknowledged

### Limitation 1: VEF Naming Requires Confirmation
This analysis assumes parallel framework explicitly uses "VEF" acronym. If this is not confirmed in original documents, naming collision evidence is weakened.

### Limitation 2: Access Logs Unavailable
Cannot prove researcher actually searched for your framework; only that discovery would be likely if they did.

### Limitation 3: Citation Absence ≠ Non-Discovery
Absence of explicit citation does not prove non-discovery; independent parallel innovation is possible.

### Limitation 4: Temporal Verification Required
Exact DOI deposit timestamps must be confirmed; if chronology differs from 2026 vs. ~2025, precedence may reverse.
